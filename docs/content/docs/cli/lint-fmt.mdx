# Linting & Formatting

Reliability is a core pillar of the Manic framework. To ensure that every project follows best practices and remains bug-free, we integrate **OXC** directy into the toolchain for lightning-fast diagnostics and formatting.

## manic lint

The `manic lint` command invokes `oxlint`, a high-performance linter that is up to 100x faster than ESLint.

```bash
manic lint
```

### Why oxlint?
Legacy linters often become the bottleneck in CI/CD pipelines. Manic chooses `oxlint` because:
- **No Configuration Required**: It comes with sensible defaults for React and TypeScript out of the box.
- **Actionable Diagnostics**: Provides clear, high-fidelity error messages that help you fix issues faster.
- **Safety**: Specifically catches common React pitfalls like incorrect Hook usage or missing dependencies.

### Build Guard
By default, [manic build](/docs/cli/build) executes a mandatory lint pass. If critical errors are found, the build will terminate. This ensures that broken code never reaches your production users.

## manic fmt

For code consistency, Manic provides the `manic fmt` command, powered by the OXC formatter.

```bash
manic fmt [options]
```

### Features
- **Consistency**: Enforces a unified coding style across your entire project.
- **Blazing Fast**: Formats thousands of lines in milliseconds, making it ideal for pre-commit hooks.
- **Compatibility**: Designed to be highly compatible with Prettier's default rules, reducing the friction of migration.

### Usage Options
- `--write`: (Default) Overwrites files with the formatted version.
- `--check`: Returns a non-zero exit code if any files are not formatted.

## Configuration Integration

Manic aims for zero-config, but we understand the need for customization.
- **Linter**: Place an `.oxlintrc.json` in your project root to customize rules.
- **Formatter**: Manic respects `.prettierrc` files for basic formatting options to ensure harmony with existing team workflows.

```json
// .oxlintrc.json
{
  "rules": {
    "no-console": "off"
  }
}
```
