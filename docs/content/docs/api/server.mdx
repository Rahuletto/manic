# Server API

The Server API allows you to interact with the underlying Hono instance, manage link headers, and inject custom HTML during the server-side rendering process.

## Hooks

### `configureServer(ctx)`

Used in plugins to hook into the dev and production server lifecycle.

```ts twoslash
//
import { createPlugin } from 'manicjs/config';

export default createPlugin({
  name: 'custom-server',
  configureServer(ctx) {
    // Register a raw Request -> Response route
    ctx.addRoute('/health', () => {
      return new Response(JSON.stringify({ ok: true }), {
        headers: { 'content-type': 'application/json' },
      });
    });
  }
});
```

## Headers & Metadata

### `addLinkHeader`

```ts
ctx.addLinkHeader('</style.css>; rel=preload; as=style');
```

This is primarily used for resource preloading and discovery (e.g. MCP discovery).

### `injectHtml`

```ts
ctx.injectHtml('<meta name="manic-id" content="123">');
```

Injected tags are appended to the `<head>` segment only once per request.

## Server Components

### `<ServerError />`

The default error boundary for server-side exceptions. You can customize the look and feel using CSS variables:

- `--theme-background`
- `--theme-accent`
- `--theme-foreground`
