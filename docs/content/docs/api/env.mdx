# Environment Variables

Manic uses the `MANIC_PUBLIC_` prefix to expose environment variables to the client-side bundle. All other variables remain server-only.

## Client-Side API

### `getEnv()`

Gets a single environment variable value. Only variables prefixed with `MANIC_PUBLIC_` are accessible on the client.

```ts twoslash
// Client-side access to public variables
declare function getEnv(key: string): string | undefined;

const apiUrl = getEnv('MANIC_PUBLIC_API_URL');
// Returns: "https://api.example.com" or undefined
```

#### Parameters

<TypeTable
  type={{
    key: {
      type: 'string',
      description: 'Environment variable name (must start with MANIC_PUBLIC_)',
    },
  }}
/>

#### Return Value

```ts
function getEnv(key: string): string | undefined;
```

#### Examples

```ts twoslash
// Client-side: accessing multiple public variables
declare function getEnv(key: string): string | undefined;

function getApiConfig() {
  const apiUrl = getEnv('MANIC_PUBLIC_API_URL');
  const analyticsId = getEnv('MANIC_PUBLIC_ANALYTICS_ID');
  
  return { apiUrl, analyticsId };
}
```

### `getPublicEnv()`

Gets all public environment variables as an object.

```ts twoslash
// Client-side: get all public environment variables
declare function getPublicEnv(): { [key: string]: string };

const env = getPublicEnv();
// Returns: { MANIC_PUBLIC_API_URL: "...", MANIC_PUBLIC_ANALYTICS_ID: "..." }
```

#### Return Value

```ts
interface PublicEnv {
  [key: string]: string;
}

function getPublicEnv(): PublicEnv;
```

## Server-Side Access

On the server, use standard Node.js environment variables:

```ts twoslash
// Server-side: accessing variables via process.env
const databaseUrl = process.env.DATABASE_URL;
const apiSecret = process.env.API_SECRET;

if (!databaseUrl) {
  throw new Error('DATABASE_URL is required');
}
```

In API routes:

```ts twoslash
// app/api/config/index.ts
export async function GET() {
  const dbUrl = process.env.DATABASE_URL;
  const secret = process.env.API_SECRET;
  
  return Response.json({
    db: dbUrl ? 'connected' : 'missing',
  });
}
```

## Environment Files

Manic loads environment files automatically:

| File | Scope | Notes |
|------|-------|-------|
| `.env` | All | Base variables |
| `.env.local` | Local | Not committed |
| `.env.development` | Dev only | Overrides `.env` |
| `.env.production` | Prod only | Overrides `.env` |

### Defining Variables

```
# .env

# Server-only (never exposed to client)
DATABASE_URL="postgres://user:pass@localhost:5432/db"
API_SECRET="super-secret-key"

# Client-accessible (exposed to browser)
MANIC_PUBLIC_API_URL="https://api.example.com"
MANIC_PUBLIC_ANALYTICS_ID="UA-XXXXX"
```



## Best Practices

<Callout type="warn">

**Never expose secrets** ŌĆö Only use `MANIC_PUBLIC_` for non-sensitive data. API keys, database passwords, and tokens must stay server-only.

</Callout>

<Callout type="info">

**Use `.env.local`** for development secrets (add to `.gitignore`).

</Callout>

<Callout type="info">

**Validate required variables** ŌĆö Check critical env vars on server startup.

</Callout>

### Example: Validating Required Variables

```ts
// ~manic.ts
import { createManicServer } from 'manicjs/server';
import app from './app/index.html';

const requiredEnv = ['DATABASE_URL', 'API_SECRET'];

for (const key of requiredEnv) {
  if (!process.env[key]) {
    throw new Error(`Missing required environment variable: ${key}`);
  }
}

const server = createManicServer({
  html: app,
});

export default server;
```
