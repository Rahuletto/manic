# Client

This page covers client-side patterns available in Manic.

## Fetch Pattern

The recommended way to call APIs from the client:

```ts
// Fetch data from API
const res = await fetch('/api/users');
const data = await res.json();
```

## Environment Variables (Client-Side)

Access public environment variables in the browser:

```ts
const apiUrl = getEnv('MANIC_PUBLIC_API_URL');
```

See [Environment Variables](/docs/framework/advanced/environment-variables) for full documentation.

## Client-Side Only Code

Use environment checks for client-specific logic:

```ts
// Only run on client
if (typeof window !== 'undefined') {
  localStorage.setItem('key', 'value');
}

// Use Bun.env for build-time environment
const isDev = Bun.env.NODE_ENV === 'development';
```

## Type-Safe API Calls

Share types with your API routes for full type-safety:

```ts
// app/api/users/index.ts - Export the type
import { Hono } from 'hono';

const app = new Hono();
app.get('/', (c) => c.json({ users: [] }));
export type AppType = typeof app;
```

Then use in client code:

```ts
import type { AppType } from '../api/users';
import { hc } from 'hono/client';

const client = hc<AppType>('/api');
const { users } = await client.users.$get();
```
