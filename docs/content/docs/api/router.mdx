# Router Reference

The Manic Router is a zero-config, type-safe React routing engine built for SPAs. It manages navigation state, handles code-splitting, and orchestrates View Transitions.

## Components

### `<Link />`

The primary component for client-side navigation.

```tsx twoslash
import React from 'react';
import { Link } from 'manicjs';

<Link to="/posts/123" prefetch={true} replace={false}>
  View Post
</Link>
```

#### Props

| Prop | Type | Default | Description |
| :--- | :--- | :--- | :--- |
| `to` | `string` | **Required** | The destination URL path. |
| `prefetch` | `boolean` | `true` | Preloads the target route's code chunk on hover/focus. |
| `replace` | `boolean` | `false` | Replaces the current history entry instead of pushing a new one. |
| `viewTransition` | `boolean` | `true` | Wraps the navigation in `document.startViewTransition`. |
| `className` | `string` | `undefined` | Standard CSS class name. |

## Hooks

### `useRouter()`

Provides access to the router instance for programmatic navigation.

```tsx twoslash
import React from 'react';
import { useRouter } from 'manicjs';

function MyComponent() {
  const router = useRouter();
  
  // Navigate to a new route
  const goToHome = () => router.navigate('/');
  
  // Replace current route
  const reset = () => router.navigate('/login', { replace: true });
  
  return <button onClick={goToHome}>Home</button>;
}
```

#### Return Type
```ts
interface RouterContextValue {
  path: string;
  navigate(to: string, options?: { replace?: boolean }): void;
  params: Record<string, string>;
}
```

### `useParams()`

Returns an object of the dynamic segments from the current URL.

```tsx
// For app/routes/blog/[slug].tsx
const { slug } = useParams();
```

### `useQueryParams()`

A hook to manage URL search parameters with a React-state-like API.

```tsx
const [query, setQuery] = useQueryParams();

// Access: query.get('q')
// Update: setQuery({ q: 'new search' })
```

## Internal: Route Manifest

The router relies on a generated manifest located at `app/~routes.generated.ts`. This file is automatically updated by the `manic dev` and `manic build` processes. **Do not edit this file manually.**
