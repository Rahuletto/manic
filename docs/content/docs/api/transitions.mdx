# View Transitions

Manic exposes a small wrapper around the browser's native [View Transitions API](https://developer.mozilla.org/en-US/docs/Web/API/View_Transitions_API). When `router.viewTransitions` is enabled (default), every navigation through `<Link>` or `navigate()` is wrapped in a `document.startViewTransition()` call automatically.

The `ViewTransitions` namespace lets you mark elements that should animate **across** navigations.

```ts twoslash
import { ViewTransitions } from 'manicjs';
```

---

## Tagging Elements

Each property on `ViewTransitions` corresponds to an HTML element. Pass a `name` prop and Manic injects the matching `viewTransitionName` CSS property:

```tsx twoslash
// @errors: 2322
import React from 'react';
import { Link, ViewTransitions } from 'manicjs';

export function ProductCard({ id, image, title }: { id: string; image: string; title: string }) {
  return (
    <Link to={`/products/${id}`}>
      <ViewTransitions.img name={`product-${id}`} src={image} alt={title} />
      <ViewTransitions.h2 name={`product-title-${id}`}>{title}</ViewTransitions.h2>
    </Link>
  );
}
```

On the destination page, render an element with the **same** `name` and the browser will animate the transition between the two:

```tsx twoslash
// @errors: 2322
// app/routes/products/[id].tsx
import React from 'react';
import { useRouter, ViewTransitions } from 'manicjs';

export default function ProductPage() {
  const { params } = useRouter();
  return (
    <article>
      <ViewTransitions.img name={`product-${params.id}`} src="/full.jpg" alt="" />
      <ViewTransitions.h1 name={`product-title-${params.id}`}>Product details</ViewTransitions.h1>
    </article>
  );
}
```

## Supported Tags

| Category | Elements |
| :--- | :--- |
| Layout | `div`, `span`, `main`, `section`, `article`, `header`, `footer`, `nav`, `aside` |
| Headings | `h1`, `h2`, `h3` |
| Content | `p`, `img`, `button`, `a`, `ul`, `li` |

Each accepts the same props as the native HTML element plus a required `name` string.

## Customizing the Animation

The browser uses CSS pseudo-elements for transitions. Style them globally:

```css title="app/global.css"
::view-transition-old(root),
::view-transition-new(root) {
  animation-duration: 240ms;
  animation-timing-function: cubic-bezier(0.2, 0, 0, 1);
}

::view-transition-old(product-123) {
  animation: fade-out 240ms both;
}

::view-transition-new(product-123) {
  animation: fade-in 240ms both;
}
```

Names containing dynamic IDs (`product-${id}`) require the same dynamic IDs in CSS, or you can target a stable group with `view-transition-class`.

---

## `setViewTransitions(enabled)`

Toggle the API on or off at runtime. This is useful when integrating with motion-sensitivity preferences:

```ts
// @errors: 2322
import { setViewTransitions } from 'manicjs/transitions';

if (matchMedia('(prefers-reduced-motion: reduce)').matches) {
  setViewTransitions(false);
}
```

You can also disable transitions globally with the configuration option:

```ts title="manic.config.ts"
import { defineConfig } from 'manicjs/config';

export default defineConfig({
  router: { viewTransitions: false },
});
```

---

## Browser Support

The View Transitions API is supported in Chromium-based browsers and Safari (Tech Preview). When the API is unavailable, Manic falls back to an instant DOM swap with no animation — your application keeps working without any code changes.
