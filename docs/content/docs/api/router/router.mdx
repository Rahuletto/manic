# `<Router />`

The **`Router`** component is the client root for Manic file-based routing. It reads lazy loaders from **`window.__MANIC_ROUTES__`** (injected when your **`main`** entry imports **`./app/~routes.generated`**) or accepts an explicit **`routes`** map for testing.

```tsx
// app/main.tsx ŌĆö typical wiring (~routes.generated exists after discovery/build only)
import { createRoot } from 'react-dom/client';
import { Router } from 'manicjs';
import { routes, notFoundPage, errorPage } from './~routes.generated';

window.__MANIC_ROUTES__ = routes;
window.__MANIC_ERROR_PAGES__ = {};
if (notFoundPage) window.__MANIC_ERROR_PAGES__.notFound = notFoundPage;
if (errorPage) window.__MANIC_ERROR_PAGES__.error = errorPage;

createRoot(document.getElementById('root')!).render(<Router />);
```

You may pass **`routes`** explicitly instead of using **`window.__MANIC_ROUTES__`** (useful in tests):

```tsx
<Router routes={routes} />
```

**Not-found** and **route error** UI come from **`window.__MANIC_ERROR_PAGES__`** (`.notFound` and `.error` lazy loaders), normally assigned alongside **`__MANIC_ROUTES__`** in **`app/main.tsx`**.

## Behavior (framework context)

| Concern | Detail |
| :--- | :--- |
| **Matching** | Uses internal **`RouteRegistry`** ŌĆö same precedence as discovery (static beats dynamic beats catchŌĆæall). |
| **Navigation** | **`navigate`** / **`<Link>`** update history and swap the lazy component; optional View Transitions when enabled. |
| **Prefetch** | **`preloadRoute`** / **`Link`** hover prefetch warm the same loaders. |
| **Errors** | Route render failures surface **`ErrorOverlay`** in development patterns; **`ServerError`** shell exists for fatal client paths ŌĆö pair with **`~500.tsx`** for branded pages ([Errors overview](/docs/api/errors)). |
