# OXC Toolchain

Manic replaces traditional JavaScript tools (like Babel, Terser, or ESLint) with the **OXC** (The Oxidation Compiler) toolchain. Written in Rust, OXC provides a massive performance boost to our build pipeline.

## Transpilation

We use `oxc-transform` for all JSX and TypeScript compilation. This allows Manic to:
- Transpile code 10-100x faster than traditional tools.
- Maintain modern ES target compatibility (defaults to ES2022).
- Preserve React Fast Refresh semantics for a seamless dev experience.

## Minification

In production builds, `oxc-minify` handles the compression of our client and server bundles. It performs:
- **Dead code elimination**: Removing unreachable code.
- **Mangle symbols**: Shortening variable and function names.
- **Constant folding**: Pre-calculating static expressions.

## Linting

The `manic lint` command triggers `oxlint`, which scans your codebase for potential bugs and performance regressions in milliseconds. It is configured via `.oxlintrc.json` and ensures production-grade code quality without the overhead of ESLint.

## Integration

OXC is integrated into Manic's CLI as a Bun plugin during development and as a post-processing step during `manic build`.
