# Project Structure

Manic follows a simple, convention-over-configuration directory structure.

## Overview

A typical Manic project looks like this:

<Files>
  <Folder name="app" defaultOpen>
    <Folder name="api">
      <File name="hello/index.ts" />
    </Folder>
    <Folder name="routes">
      <File name="index.tsx" />
      <File name="about.tsx" />
    </Folder>
    <File name="global.css" />
    <File name="index.html" />
    <File name="main.tsx" />
  </Folder>
  <Folder name="packages" />
  <File name="manic.config.ts" />
  <File name="~manic.ts" />
  <File name="package.json" />
  <File name="tsconfig.json" />
</Files>

## Core Directories

### `app/`
This is where your application logic lives.

- **`app/routes/`**: Every `.tsx` file in this directory (except those prefixed with `~`) becomes a public page. For example, `app/routes/about.tsx` maps to `/about`.
- **`app/api/`**: Every subdirectory with an `index.ts` file becomes an API endpoint. For example, `app/api/hello/index.ts` maps to `/api/hello`.
- **`app/main.tsx`**: The entry point for the React client.

### `packages/`
If your project is a monorepo or contains shared utilities, you can place them here.

## Configuration

### `manic.config.ts`
The main configuration file where you define plugins, providers, and framework settings.

### `~manic.ts`
The server entry point. This file initializes the Hono server and loads the application. The `~` prefix indicates that it is an internal framework file.
