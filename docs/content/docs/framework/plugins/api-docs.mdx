# API Docs

Mounts an interactive [Scalar](https://scalar.com) API reference UI that renders an OpenAPI spec.

## Installation

```bash
bun add @manicjs/api-docs
```

## Usage

```ts
// manic.config.ts
import { defineConfig } from 'manicjs/config';
import { apiDocs } from '@manicjs/api-docs';

export default defineConfig({
  plugins: [
    apiDocs(),
  ],
});
```

This serves the Scalar docs UI at `/docs`, loading the OpenAPI spec from `/openapi.json`.

<Callout type="info">

The plugin does **not** generate an OpenAPI spec. You must provide the spec yourself — for example, via an API route that returns your spec as JSON at `/openapi.json`.

</Callout>

## Options

<TypeTable
  type={{
    path: {
      type: 'string',
      default: '"/docs"',
      description: 'Path to mount the docs UI.',
    },
    specUrl: {
      type: 'string',
      default: '"/openapi.json"',
      description: 'URL of the OpenAPI spec to load.',
    },
    theme: {
      type: 'string',
      default: '"default"',
      description: 'Scalar theme name.',
    },
  }}
/>

## Custom Configuration

```ts
export default defineConfig({
  plugins: [
    apiDocs({
      path: '/api-reference',
      specUrl: '/api/spec.json',
      theme: 'modern',
    }),
  ],
});
```

## Limitations

<Callout type="warn">

This plugin only has a `configureServer` hook — it does **not** include a `build` hook. The docs UI is available in development but is not included in production builds.

</Callout>
