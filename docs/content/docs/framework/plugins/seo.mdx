# SEO

Global SEO defaults (meta tags, Open Graph, Twitter Cards, robots.txt) plus a `<Metadata>` component for per-page overrides.

## Installation

```bash
bun add @manicjs/seo
```

## Quick Start

### 1. Configure Global Defaults

```ts
// manic.config.ts
import { defineConfig } from 'manicjs/config';
import { seo } from '@manicjs/seo';

export default defineConfig({
  plugins: [
    seo({
      hostname: 'https://example.com',
      title: 'My App',
      description: 'Build amazing apps with Manic',
    }),
  ],
});
```

### 2. Override Per Page

```tsx
// app/routes/about.tsx
import { Metadata } from '@manicjs/seo/metadata';

export default function About() {
  return (
    <>
      <Metadata
        title="About Us"
        description="Learn more about our company"
        ogImage="/images/about-og.png"
      />
      <h1>About Us</h1>
    </>
  );
}
```

The `<Metadata>` component overrides the global defaults for that page. When the user navigates away, the previous values are restored automatically.

## Full Example

```ts
// manic.config.ts
import { defineConfig } from 'manicjs/config';
import { seo } from '@manicjs/seo';

export default defineConfig({
  plugins: [
    seo({
      hostname: 'https://example.com',
      title: 'My App',
      description: 'Build amazing apps with Manic',
      author: 'John Doe',
      twitter: {
        card: 'summary_large_image',
        site: '@myapp',
        creator: '@johndoe',
      },
      openGraph: {
        type: 'website',
        locale: 'en_US',
        image: 'https://example.com/og.png',
        siteName: 'My App',
      },
      rules: [
        { userAgent: '*', allow: ['/'], disallow: ['/admin'] },
      ],
      contentSignals: {
        'ai-train': 'no',
        search: 'yes',
      },
    }),
  ],
});
```

## Per-Page Metadata

The `<Metadata>` component lets you override global SEO defaults on individual pages. It renders nothing visible — it manages `<head>` tags via side effects.

```tsx
import { Metadata } from '@manicjs/seo/metadata';
```

### Usage Examples

#### Static Metadata

```tsx
// app/routes/about.tsx
import { Metadata } from '@manicjs/seo/metadata';

export default function About() {
  return (
    <>
      <Metadata
        title="About Us"
        description="Learn more about our company"
        canonical="https://example.com/about"
      />
      <h1>About Us</h1>
    </>
  );
}
```

#### Dynamic Metadata

Use variables and state to set metadata dynamically:

```tsx
// app/routes/blog/[slug].tsx
import { Metadata } from '@manicjs/seo/metadata';
import { useRouter } from 'manicjs';

const posts = {
  'hello-world': { title: 'Hello World', excerpt: 'My first post', image: '/images/hello.png' },
  'react-tips': { title: 'React Tips', excerpt: 'Useful React patterns', image: '/images/react.png' },
};

export default function BlogPost() {
  const { params } = useRouter();
  const post = posts[params.slug];

  if (!post) return <div>Post not found</div>;

  return (
    <>
      <Metadata
        title={post.title}
        description={post.excerpt}
        ogImage={post.image}
        ogType="article"
      />
      <article>
        <h1>{post.title}</h1>
        <p>{post.excerpt}</p>
      </article>
    </>
  );
}
```

#### With Fetched Data

```tsx
import { Metadata } from '@manicjs/seo/metadata';
import { useState, useEffect } from 'react';

export default function Product() {
  const [product, setProduct] = useState(null);

  useEffect(() => {
    fetch('/api/product/1').then(r => r.json()).then(setProduct);
  }, []);

  return (
    <>
      {product && (
        <Metadata
          title={product.name}
          description={product.description}
          ogImage={product.image}
        />
      )}
      <div>{product?.name ?? 'Loading...'}</div>
    </>
  );
}
```

### How It Works

1. On mount, `<Metadata>` updates existing `<meta>` / `<link>` tags or creates new ones in `<head>`
2. It saves the previous values of any tags it modifies
3. On unmount (when navigating away), it restores the original values — so global SEO defaults come back automatically
4. Tags created by `<Metadata>` that didn't exist before are removed on cleanup

## Integration with Sitemap

Works with `@manicjs/sitemap` for complete SEO:

```ts
import { defineConfig } from 'manicjs/config';
import { seo } from '@manicjs/seo';
import { sitemap } from '@manicjs/sitemap';

export default defineConfig({
  plugins: [
    seo({
      hostname: 'https://example.com',
      title: 'My App',
    }),
    sitemap({
      hostname: 'https://example.com',
    }),
  ],
});
```

When `autoSitemap` is `true` (the default), the SEO plugin automatically adds `/sitemap.xml` to `robots.txt` and emits a `Link` header for sitemap discovery.
