# UnoCSS

Integrates UnoCSS for atomic CSS styling in Manic. An alternative to Tailwind CSS with more flexibility.

## Installation

```bash
bun add @manicjs/unocss
```

## Quick Start

### 1. Add the Plugin

```ts twoslash
// @errors: 2322
// manic.config.ts
import { defineConfig } from 'manicjs/config';
declare function unocss(options?: Record<string, unknown>): { name: string };

export default defineConfig({
  plugins: [unocss()],
});
```

The `unocss()` plugin takes no options. It registers `bun-plugin-unocss` which auto-detects your UnoCSS configuration from the project root.

### 2. Create `uno.config.ts`

All UnoCSS configuration lives in `uno.config.ts` at your project root:

```ts twoslash
// @errors: 2322
// uno.config.ts
declare function defineConfig(options: Record<string, unknown>): { presets: any[] };
declare function presetUno(): { name: string };
declare function presetAttributify(options?: Record<string, unknown>): { name: string };

export default defineConfig({
  presets: [
    presetUno(),
    presetAttributify(),
  ],
});
```

## UnoCSS Presets

### Core Presets

```ts twoslash
// @errors: 2322
// uno.config.ts
declare function defineConfig(options: Record<string, unknown>): { presets: any[] };
declare function presetUno(): { name: string };
declare function presetIcons(options?: Record<string, unknown>): { name: string };
declare function presetWebFonts(): { name: string };

export default defineConfig({
  presets: [
    presetUno(),           // Core utilities
    presetIcons(),         // Icon set
    presetWebFonts(),      // Google Fonts
  ],
});
```

### Attributify Mode

```ts twoslash
// @errors: 2322
// uno.config.ts
declare function defineConfig(options: Record<string, unknown>): unknown;
declare function presetUno(): unknown;
declare function presetAttributify(options?: Record<string, unknown>): unknown;

export default defineConfig({
  presets: [
    presetUno(),
    presetAttributify({
      /* attributify options */
    }),
  ],
});
```

Then use in JSX:

```tsx twoslash
// @errors: 2322
<>
  <div class="flex gap-2" />
  <div flex gap-2 />       {/* attributify */}
  <div bg-red-500 />        {/* color shortcuts */}
</>
```

## Custom Rules

```ts twoslash
// @errors: 2322
// uno.config.ts
declare function defineConfig(options: Record<string, unknown>): unknown;
declare function presetUno(): unknown;

export default defineConfig({
  presets: [presetUno()],
  rules: [
    [/^text-(.*)$/, ([c]: [string]) => ({ color: c })],
    ['custom-shadow', { 'box-shadow': '0 4px 6px rgba(0,0,0,0.1)' }],
  ],
});
```

## Shortcuts

Define reusable class combinations:

```ts twoslash
// @errors: 2322
// uno.config.ts
declare function defineConfig(options: Record<string, unknown>): unknown;
declare function presetUno(): unknown;

export default defineConfig({
  presets: [presetUno()],
  shortcuts: [
    ['btn', 'px-4 py-2 rounded bg-blue-500 text-white hover:bg-blue-600'],
    ['input-field', 'w-full px-3 py-2 border border-gray-300 rounded'],
  ],
});
```

Usage:

```tsx twoslash
// @errors: 2322
<>
  <button class="btn">Click</button>
  <input class="input-field" />
</>
```

## Icon Sets

```ts twoslash
// @errors: 2322
// uno.config.ts
declare function defineConfig(options: Record<string, unknown>): unknown;
declare function presetUno(): unknown;
declare function presetIcons(options?: Record<string, unknown>): unknown;

export default defineConfig({
  presets: [
    presetUno(),
    presetIcons({
      scale: 1.2,
      cdn: 'https://esm.sh/',
    }),
  ],
});
```

```tsx twoslash
// @errors: 2322
<>
  <i class="i-carbon-sun text-xl" />
  <i class="i-carbon-moon text-xl" />
</>
```

## Theme Integration

```ts twoslash
// @errors: 2322
// uno.config.ts
declare function defineConfig(options: Record<string, unknown>): unknown;
declare function presetUno(): unknown;

export default defineConfig({
  presets: [presetUno()],
  theme: {
    colors: {
      primary: '#3b82f6',
      secondary: '#8b5cf6',
    },
  },
});
```

## Vs Tailwind CSS

| Feature | UnoCSS | Tailwind CSS |
|---------|--------|-------------|
| Bundle size | Smaller (atomic only) | Full framework |
| Customization | More flexible | Fixed methodology |
| Icon system | Built-in | Requires plugin |
| Attributify mode | Native | No |
| Virtual modules | Yes | No |
