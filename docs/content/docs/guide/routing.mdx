# Routing

Manic features a file-system-based router that converts your `app/routes/` directory into a highly optimized runtime registry. It supports nested layouts, dynamic segments, catch-all routes, and the native **View Transitions API**.

## File System Routing

Every file in `app/routes/` with a `.tsx` extension satisfies a route path.

| File Path | URL Path | Description |
| :--- | :--- | :--- |
| `app/routes/index.tsx` | `/` | Home page |
| `app/routes/about.tsx` | `/about` | Static route |
| `app/routes/posts/[id].tsx` | `/posts/123` | Dynamic route |
| `app/routes/docs/[:...slug].tsx` | `/docs/a/b/c` | Catch-all route |

> [!TIP]
> Manic supports both colon syntax (`:param`) and bracket syntax (`[param]`) for dynamic routes, allowing you to choose the convention that fits your preferences.

## Route Scoring & Priority

When multiple routes could match a single URL, Manic uses a scoring algorithm to determine the winner. This ensures that static routes always take precedence over dynamic ones.

### Scoring Rules
- **Static Segments**: +100 points per segment.
- **Dynamic Segments**: +10 points per segment.
- **Catch-all Segments**: +1 point per segment.

**Example Conflict:**
- `app/routes/posts/new.tsx` (Score: 200) wins over...
- `app/routes/posts/[id].tsx` (Score: 110)

This logic is implemented in the `RouteRegistry` to ensure O(n) matching performance even with thousands of routes.

## The `~` Convention (Private Files)

You can exclude files and folders from the router by prefixing them with a tilde (`~`). This is the recommended way to colocate components, hooks, and tests directly within your routes directory.

```text
app/routes/
  index.tsx         <-- Route (/)
  ~Header.tsx       <-- Ignored (Component)
  ~hooks/           <-- Ignored (Folder)
    useAuth.ts
```

## Navigation & Prefetching

Manic provides the `<Link>` component for client-side navigation. 

```tsx twoslash
import React from 'react';
import { Link } from 'manicjs';

export default function Nav() {
  return (
    <Link to="/dashboard" prefetch>
      Dashboard
    </Link>
  );
}
```

### Automatic Code-Splitting
When you use `<Link>`, Manic automatically:
1.  **Observes**: Monitors the link visibility or hover state.
2.  **Prefetches**: Downloads the target route's JavaScript chunk *before* the user clicks.
3.  **Transitions**: Uses `document.startViewTransition` (if supported) to provide a fluid, app-like navigation experience.

## Programmatic Navigation

For logic-based navigation, use the `useRouter` hook.

```tsx twoslash
import React from 'react';
import { useRouter } from 'manicjs';

export function LoginButton() {
  const router = useRouter();

  const handleLogin = async () => {
    // ... login logic
    router.navigate('/dashboard');
  };

  return <button onClick={handleLogin}>Login</button>;
}
```

## Catch-all Routes

Catch-all routes are useful for documentation sites or CMS integrations where the path depth is unknown.

- **File**: `app/routes/docs/[...slug].tsx`
- **URL**: `/docs/getting-started/installation`
- **Params**: `{ slug: "getting-started/installation" }`
